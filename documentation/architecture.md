# АРХИТЕКТУРА ПРОЕКТА AGS_PROJ
============================

## ОБЩЕЕ ОПИСАНИЕ
--------------
AGS_PROJ - это модульный парсер веб-страниц с функцией детекции AI-генерированного контента. 
Проект спроектирован по принципу разделения ответственности: каждый модуль отвечает за 
конкретную задачу.

## ОСНОВНЫЕ КОМПОНЕНТЫ
-------------------

### 1. main.py (Точка входа)
   Назначение: Оркестрация всего процесса работы приложения
   Отвечает за:
   - Загрузку конфигурации
   - HTTP-запросы к веб-страницам
   - Координацию парсинга и анализа
   - Сохранение результатов
   
   Ключевые элементы:
   - Класс Config: инкапсулирует работу с config.json, предоставляет доступ к параметрам
   - get_requester_page(): загрузка и парсинг HTML страницы
   - get_comments(): извлечение заголовка и комментариев
   - dump_json(): сериализация данных
   - list_low_entropy(): фильтрация комментариев по энтропии

### 2. detector.py (Модуль анализа текста)
   Назначение: Вычисление метрик текста для детекции AI-генерации
   Отвечает за:
   - Токенизацию русского текста (библиотека razdel)
   - Расчет энтропии Шеннона
   - Предоставление метрики "естественности" текста
   
   Ключевые функции:
   - russian_entropy(text): вычисляет энтропию текста
     Формула: H = -Σ p(x) * log2(p(x))
     где p(x) - вероятность встречи токена
   
   - is_ai_russian(text): возвращает значение энтропии
     Низкая энтропия (< 4.0) указывает на возможную AI-генерацию
     Высокая энтропия характерна для человеческого текста

### 3. config.json (Конфигурация)
   Назначение: Централизованное хранение настраиваемых параметров
   Содержит:
   - url: адрес целевой страницы
   - comment_tree_class: CSS-класс контейнера комментариев
   - comment_text_class: CSS-класс текста комментария
   - ai_enthropy_threshold: порог для детекции AI

## ПОТОК ДАННЫХ
------------
1. Запуск main.py
   ↓
2. Загрузка конфигурации (Config.__init__)
   ↓
3. HTTP GET запрос к URL (get_requester_page)
   ↓
4. Парсинг HTML через BeautifulSoup
   ↓
5. Извлечение комментариев (get_comments)
   ↓
6. Анализ энтропии каждого комментария (detector.is_ai_russian)
   ↓
7. Вывод комментариев с низкой энтропией
   ↓
8. Сохранение всех данных в output.json

## ЗАВИСИМОСТИ И ОБОСНОВАНИЕ ВЫБОРА
---------------------------------
- requests (>=2.31.0): стандарт де-факто для HTTP-запросов в Python
- beautifulsoup4 (>=4.12.0): мощный и гибкий парсер HTML/XML
- html5lib (>=1.1): парсер, создающий валидное HTML5 дерево
- razdel (>=0.5.0): специализированная библиотека для токенизации русского текста
  (быстрее и точнее для русского языка, чем nltk/spaCy)

## АРХИТЕКТУРНЫЕ РЕШЕНИЯ
---------------------
### 1. Разделение конфигурации и кода
   Почему: позволяет менять параметры без изменения кода
   Преимущество: гибкость, возможность работы с разными сайтами

### 2. Использование энтропии Шеннона
   Почему: AI-тексты часто имеют более предсказуемую структуру
   Преимущество: простой и быстрый метод детекции без ML-моделей
   Ограничение: не всегда точен, требует настройки порога

### 3. Модульная структура
   Почему: detector.py выделен в отдельный модуль
   Преимущество: переиспользуемость, тестируемость

### 4. Класс Config вместо глобальных переменных
   Почему: инкапсуляция, возможность расширения
   Преимущество: чистый код, легче тестировать

## ТОЧКИ РАСШИРЕНИЯ
----------------
### 1. Добавление новых источников
   - Изменить config.json (URL и CSS-селекторы)
   - При необходимости доработать get_comments()

### 2. Улучшение детекции AI
   - Добавить новые метрики в detector.py
   - Интегрировать ML-модели
   - Добавить анализ стилистических паттернов

### 3. Экспорт данных
   - Добавить новые форматы (CSV, XML) в дополнение к JSON
   - Интеграция с базами данных

### 4. Многопоточность
   - Параллельная обработка нескольких страниц
   - Асинхронные запросы (aiohttp)

## ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ
---------------------
1. Работает только с статическими страницами (не поддерживает JavaScript)
2. Порог энтропии требует ручной настройки
3. Нет обработки ошибок сети (таймауты, повторные запросы)
4. Нет валидации входных данных
5. CSS-селекторы жестко заданы для конкретной структуры Хабра

## БЕЗОПАСНОСТЬ
------------
- Не хранит чувствительные данные
- Не выполняет код с сервера
- Использует только GET-запросы
